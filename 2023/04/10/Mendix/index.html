敏捷开发

两种开发模式：

  • 瀑布模型：软件开发之前一次性定好所有的需求

  • 敏捷开发：不断迭代修改、更好的、高效的交付用户需求

敏捷开发流程

  • 客户提出用户清单需求 Backlog，整理出 User Story，选出一次迭代周期的需求清单 Sprint Backlog
  • 每次迭代的需求清单大约为期 1 到 2 周，每天 Scrum Master 会进行一个结果同步

微流

微流简介

微流可以为您的应用程序添加逻辑。 逻辑用于使您的应用程序更智能、自动化并使其可维护。

简单的按钮仅仅作为如：CreateObject、Show A Page 等默认行为，对于复杂的业务可以使用微流进行处理。

微流的作用：

  • 扩展或改变默认行为
  • 处理特殊的业务流程
  • 集成外部系统、数据库、Web 服务

新建模块 GeneralExtentions

新建模块 GeneralExtentions ，该模块放置通用的用户管理等功能。

  • 右击 App 结点
  • 选择 Add Module… 创建新模块命名为 GeneralExtentions
  • 为 GeneralExtentions 添加 2 个模块角色：Administrator、User
  • 将新的模块角色关联到相应的用户角色，将 APP 的 Administrator 分配给新模块的 Administrator 角色，将 APP 的其他角色分配给新模块的 User 角色

  • GeneralExtensions 模块

  • 大型应用拆分多个模块

Sass 入门

Sass 简介

  • Sass （英文全称：Syntactically Awesome Stylesheets） 是一个最初由 Hampton Catlin 设计并由 Natalie Weizenbaum 开发的层叠样式表语言。
  • Sass 是一个 CSS 预处理器。
  • Sass 是 CSS 扩展语言，可以帮助我们减少 CSS 重复的代码，节省开发时间。
  • Sass 完全兼容所有版本的 CSS。
  • Sass 扩展了 CSS3，增加了规则、变量、混入、选择器、继承、内置函数等等特性。
  • Sass 生成良好格式化的 CSS 代码，易于组织和维护。
  • Sass 文件后缀为 .scss。

Sass @import、Partials

Sass 可以帮助减少 CSS 重复的代码，节省开发时间。

通过安装不同的属性来创建一些代码文件，如：变量定义的文件、颜色相关的文件、字体相关的文件等。

类似 CSS，Sass 支持 @import 指令。
